# 个人网站项目 - 开发指南

> 给新手的快速上手手册。所有基础操作都在这里，遇到问题先看这个。

---

## 1. 项目位置

```
D:\projects\personal\personal_site\
```

**所有开发操作都必须在这个目录下进行。**

---

## 2. 启动项目

### 第一步：打开终端
在 VS Code 中：
- 快捷键：`Ctrl` + `` ` ``（反引号，Tab 键上面那个）
- 或菜单：`终端` → `新建终端`

### 第二步：进入项目目录
在终端里输入：
```powershell
cd D:\projects\personal\personal_site
```

### 第三步：启动开发服务器
```powershell
pnpm dev
```

成功后终端会显示：
```
VITE v6.4.2  ready in xxx ms

  Local:   http://localhost:5173/
```

### 第四步：在浏览器查看
按住 `Ctrl` 键，用鼠标**点击终端里的链接**（`http://localhost:5173/`），它会自动在浏览器打开。

## 3. 开发工作流

```
┌─────────────────────┐      保存 (Ctrl+S)      ┌─────────────────────┐
│   VS Code (写代码)   │ ──────────────────────→ │    浏览器 (看效果)   │
│                     │                         │                     │
│  改代码 → Ctrl+S    │                         │  自动刷新，无需手动  │
─────────────────────┘                         └─────────────────────┘
```

**关键点：**
- 改完代码按 `Ctrl + S` 保存。
- 浏览器会**自动刷新**显示最新效果，不用手动 F5。
- 如果有报错，错误信息会显示在 VS Code 的**终端**里，不用去浏览器找。

---

## 4. 常用命令

| 命令 | 作用 |
|------|------|
| `pnpm dev` | 启动开发服务器（开发时用） |
| `pnpm build` | 打包生产版本（上线时用） |
| `pnpm preview` | 预览打包后的效果 |
| `pnpm install` | 安装/更新依赖 |
| `pnpm add <包名>` | 安装新依赖 |
| `pnpm add -D <包名>` | 安装开发依赖 |

## 5. 文件结构说明

```
personal_site/
├── src/                    # 源代码目录（你主要在这里写代码）
│   ├── main.tsx           # 入口文件，一般不用改
│   ├── App.tsx            # 主组件，写页面内容
│   ├── index.css          # 全局样式（Tailwind 在这里导入）
│   └── vite-env.d.ts      # TypeScript 类型声明，不用管
├── index.html             # HTML 模板，一般不用改
├── vite.config.ts         # Vite 配置，一般不用改
├── package.json           # 项目配置和依赖列表
└── node_modules/          # 依赖包目录，自动生成，不用管
```

## 6. 常见问题

### Q: 终端显示 "No package.json found"
**原因**：当前目录不对。
**解决**：先 `cd D:\projects\personal\personal_site` 再运行命令。

### Q: 终端显示 "Port 5173 is in use"
**原因**：之前启动的服务没关掉，占用了端口。
**解决**：在终端按 `Ctrl + C` 停止旧服务，再重新 `pnpm dev`。

### Q: 改了代码但浏览器没变化
**解决**：
1. 确认按了 `Ctrl + S` 保存了文件。
2. 看终端有没有报错。
3. 手动刷新浏览器试试（F5）。

### Q: 怎么停止开发服务器？
**解决**：在 VS Code 终端里按 `Ctrl + C`。

### Q: 怎么关闭终端？
**解决**：终端右上角有个垃圾桶图标，点击即可。或者直接不管它，不影响写代码。

## 7. 浏览器调试技巧

按 `F12` 打开开发者工具：
- **Elements（元素）**：查看和修改 HTML/CSS。
- **Console（控制台）**：查看 JavaScript 报错。
- **手机图标**：切换手机/平板预览模式，测试响应式布局。
