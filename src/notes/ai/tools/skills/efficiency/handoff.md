# handoff — 交接

> 转自 [mattpocock/skills](https://github.com/mattpocock/skills)，版权归原作者所有。

**触发时机**：用户想把当前对话交接给另一个 agent 继续。

**描述**：将当前对话压缩为交接文档，以便另一个 agent 可以接续工作。

---

编写交接文档，总结当前对话，使全新的 agent 能够继续工作。保存到用户操作系统的临时目录——不是当前工作区。

包含"推荐技能"部分，建议下一个 agent 应该调用哪些技能。

不要重复其他工件（PRD、计划、ADR、issue、提交、diff）中已捕获的内容。改为通过路径或 URL 引用它们。

删除任何敏感信息，如 API 密钥、密码或个人身份信息。

如果用户传递了参数，将它们视为对下一个会话焦点的描述，并据此定制文档。
