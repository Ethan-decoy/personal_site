# write-a-skill — 创建技能

> 转自 [mattpocock/skills](https://github.com/mattpocock/skills)，版权归原作者所有。

**触发时机**：用户想创建、编写或构建新技能。

**描述**：创建具有正确结构、渐进式披露和配套资源的新 agent 技能。

---

## 流程

1. **收集需求** — 询问用户：
   - 技能覆盖什么任务/领域？
   - 应该处理哪些具体用例？
   - 需要可执行脚本还是仅指令？
   - 有任何要包含的参考材料吗？

2. **起草技能** — 创建：
   - SKILL.md，附带简洁指令
   - 如果内容超过 500 行，创建额外参考文件
   - 如果需要确定性操作，创建实用脚本

3. **与用户审查** — 呈现草稿并询问：
   - 这覆盖了你的用例吗？
   - 缺少什么或不清晰吗？
   - 任何部分应该更详细或更简略吗？

## 技能结构

```
skill-name/
├── SKILL.md           # 主指令（必需）
├── REFERENCE.md       # 详细文档（如需要）
├── EXAMPLES.md        # 使用示例（如需要）
└── scripts/           # 实用脚本（如需要）
    └── helper.js
```

## SKILL.md 模板

```md
---
name: skill-name
description: 能力简述。当 [具体触发条件] 时使用。
---

# 技能名称

## 快速开始

[最小工作示例]

## 工作流

[分步流程，复杂任务带检查清单]

## 高级特性

[链接到单独的文件：参见 [REFERENCE.md](REFERENCE.md)]
```

## 描述要求

描述是 **agent 在决定加载哪个技能时唯一看到的东西**。它出现在系统提示中，与所有其他已安装技能并列。你的 agent 阅读这些描述并根据用户请求选择相关技能。

**目标**：给 agent 足够的信息知道：

1. 这个技能提供什么能力
2. 何时/为什么触发它（具体关键词、上下文、文件类型）

**格式**：

- 最多 1024 字符
- 用第三人称写
- 第一句：它做什么
- 第二句："Use when [具体触发条件]"

**好示例**：

```
从 PDF 提取文本和表格、填写表单、合并文档。当处理 PDF 文件或用户提到 PDF、表单、文档提取时使用。
```

**坏示例**：

```
帮助处理文档。
```

坏示例没有给 agent 任何方式来区分这个技能和其他文档技能。

## 何时添加脚本

在以下情况添加实用脚本：

- 操作是确定性的（验证、格式化）
- 相同代码会被重复生成
- 错误需要明确处理

脚本节省 token 并提高可靠性，优于生成代码。

## 何时拆分文件

在以下情况拆分为单独文件：

- SKILL.md 超过 100 行
- 内容有明显不同的领域（金融 vs 销售 schema）
- 高级特性很少需要

## 审查检查清单

起草后确认：

- [ ] 描述包含触发条件（"Use when..."）
- [ ] SKILL.md 少于 100 行
- [ ] 无时间敏感信息
- [ ] 术语一致
- [ ] 包含具体示例
- [ ] 引用深度为一层
