# 深化方法

> 转自 [mattpocock/skills](https://github.com/mattpocock/skills)，版权归原作者所有。

如何安全地深化浅模块。

## 流程

1. **识别浅模块** — 接口几乎与实现一样复杂
2. **确认价值** — 删除测试：删除后复杂度会集中吗？
3. **设计新接口** — 小接口、大实现
4. **逐步迁移** — 保持测试通过，每次只做一件事
5. **验证深度** — 新接口是否提供真正的杠杆？

## 何时深化

- 架构审查识别出浅模块
- TDD 循环揭示测试面太窄
- 多个调用者重复相同配置模式

## 何时不深化

- 模块只是透传（删除后复杂度消失）
- 接口复杂性来自真实的外部约束
- 深化的成本超过收益
