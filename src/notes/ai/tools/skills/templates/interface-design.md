# 接口可测试性设计

> 转自 [mattpocock/skills](https://github.com/mattpocock/skills)，版权归原作者所有。

面向测试的接口设计原则。

## 原则

1. **接口决定测试面。** 好的接口让你能通过公共 API 测试所有重要行为。
2. **接缝应该是自然的。** 如果模块只能通过反射或内部手段替换协作者，那它没有好接缝。
3. **一个适配器 = 假设的接缝。两个适配器 = 真正的接缝。**

## 设计检查清单

- [ ] 模块的所有行为都能通过公共接口观察？
- [ ] 测试不需要模拟内部实现？
- [ ] 变更实现时，测试不应该失败（除非行为变了）？
- [ ] 接口暴露的是"做什么"而不是"怎么做"？
