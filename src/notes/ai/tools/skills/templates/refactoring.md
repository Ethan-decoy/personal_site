# 重构候选项

> 转自 [mattpocock/skills](https://github.com/mattpocock/skills)，版权归原作者所有。

TDD 循环后的重构信号。

## 信号

- 重复代码出现在多个文件中
- 模块接口几乎与实现一样复杂（浅模块）
- 测试需要大量 setup 才能锻炼一个简单的行为
- 变更需要同时修改 N 个紧耦合的模块
- 你为测试性提取了纯函数，但真正的逻辑在调用方式中

## 原则

- **RED 时永远不要重构。** 先到 GREEN。
- **重构前先跑测试。** 所有现有测试应该通过。
- **一次只做一件事。** 一个重构动作 → 跑测试 → 确认通过。
- **删除比移动好。** 如果你能删掉复杂度而不是重排它，那就删掉它。
