# 第0章 准备知识

> 《易筋经》是最为上乘的武功；学会了《易筋经》，你将无所不能，任你创造武功。

**计算机**是一个非常广泛的概念，所涉及的知识涵盖了众多基础学科的底层原理，我会尽可能的查漏补缺，来记录可能涉及的知识点。

## 指令集  Instruction Set

**指令集 Instruction Set**

**指令集**，顾名思义是计算机中央处理器（CPU）能够识别、解释并执行的所有机器指令的集合。

在 Computer Architecture（计算机体系结构）的专业语境下，它更准确的称谓是**指令集架构（ISA, Instruction Set Architecture）**。它并非一份简单的*操作命令清单*，而是**软件（编译器、操作系统）与硬件（微架构、物理芯片）之间严格且不可逾越的契约（Contract）**。

### 核心内涵
一个完整的 ISA 规范，必须精确界定处理器在执行程序时的所有可见要素，主要包括：
-   **指令格式与编码（Instruction Encoding）**：规定指令的二进制位排列规则。包含操作码（Opcode，指明动作）和操作数（Operands，指明对象）。分为基础**定长编码**（如 ARM/RISC-V的基础指令集，利于硬件并行译码）和**变长编码**（如 x86，代码密度高但译码逻辑复杂）。
-   **寄存器组织（Register File）**：定义处理器内部高速存储单元的数量与用途。包括**通用寄存器**（GPRs）、**程序计数器**（PC/IP，指向下一条指令）、以及记录运算状态的**状态/标志寄存器**（Flags）。
-   **数据类型与内存模型（Memory Model）**：规定支持的数据精度（如 32/64 位整数、IEEE 754 浮点数），并严格定义多核并发环境下，内存读写操作的可见性与顺序性（如 x86 的强顺序模型 TSO，或 ARM 的弱顺序模型）。
-   **异常与中断机制（Exceptions & Interrupts）**：标准化处理同步错误（如除零、缺页异常 Page Fault）和异步硬件事件（如外部设备中断）的捕获、上下文保存与恢复流程。

### 历史发展
> 指令集的发展史，本质上是一部“向前兼容的商业妥协”与“追求极致效率的现代工程美学”之间的博弈史。

*   **CISC 的架构包袱**：
    以 x86 为代表的复杂指令集（CISC），自 1978 年诞生以来，为了保住庞大的软件生态，确立了 **“绝对的二进制向前兼容”原则**。这意味着 40 多年前为 8086 编写的代码，理论上仍能在最新的处理器上运行。这种商业上的巨大成功，带来了显著的工程挑战（如解码复杂度与功耗开销，业界常称为‘解码税’）；寄存器命名保留了浓厚的“考古学”痕迹（如 AL/AX/EAX/RAX 的嵌套）；繁杂的寻址模式让指令格式充满特例。
*   **RISC 的现代工程美学**：
    相比之下，诞生于 20 世纪 80 年代的 RISC（精简指令集，如 ARM、MIPS、RISC-V）是站在更高抽象层次上的重新设计。它基于对实际程序行为的统计学分析（80/20 法则），果断砍掉了硬件实现复杂但极少使用的指令。RISC 严格遵循 **Load/Store 架构**（仅专用指令可访问内存，运算全在寄存器间进行），采用定长指令，将复杂性上移交给现代编译器。这种设计完美契合了现代 CPU 深流水线、超标量和乱序执行的需求，体现了“如无必要，勿增实体”的现代工程美学。
*   **现代微架构**：
    今天，两大阵营在顶级芯片设计中达成了奇妙的融合。x86 处理器通过引入**微操作（$\mu$ops）转换机制**，在前端将复杂的 CISC 指令“打碎”成类似 RISC 的定长微操作，再送入后端的 RISC 核心执行，实现了“外 CISC，内 RISC”。而 RISC 架构（如 ARM）为了弥补代码密度较低的缺点，也适度引入了复杂指令（如批量加载/存储）或压缩指令集（Thumb），展现了工程上的务实妥协。

### 演进趋势
现代 ISA 已不再是静态的规范，而是演变为 **基础指令集（Base ISA） + 扩展指令集（Extensions）** 的模块化设计，以应对日益细分的算力需求：

*   **SIMD 与数据级并行扩展**：
    为应对音视频编解码、科学计算与 AI 推理，ISA 引入了单指令多数据流（SIMD）扩展。例如 x86 的 AVX-512、ARM 的 SVE（可伸缩向量扩展）以及 RISC-V 的 V 扩展。它们允许一条指令同时对多个数据元素进行并行运算，提供数量级的性能提升。
*   **硬件级虚拟化与安全扩展**：
    随着云计算和机密计算的普及，ISA 必须提供底层的安全隔离机制。例如 ARM 的 EL2（异常级别 2）与 TrustZone、x86 的 VT-x 与 TDX/SGX（可信域/软件防护扩展），以及 RISC-V 的 H 扩展（Hypervisor，虚拟化支持） 与 PMP（物理内存保护）。这些扩展在硬件层面划分了安全与非安全执行环境，大幅降低了虚拟机上下文切换的开销，并保障了密钥等敏感数据的安全。
*   **开源化与领域专用架构（DSA）的定制**：
    以 RISC-V 为代表的新兴 ISA，打破了传统 x86/ARM 的封闭授权壁垒。其最大的革命性优势在于**支持自定义指令（Custom Extensions）**。芯片设计者可以在不破坏基础兼容性的前提下，为特定的算法（如专用加密、神经网络加速器）添加专属的硬件指令。这种“乐高式”的模块化演进，正成为后摩尔定律时代，突破通用计算性能瓶颈的关键路径。
