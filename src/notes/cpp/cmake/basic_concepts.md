# Basic concepts

## 编译流程

> 从源码到可执行程序，经典分为四个阶段：预处理 -> 编译 -> 汇编 -> 链接

### 预处理阶段 (Preprocessing)

- **核心任务：** 处理所有以<kbd>#</kbd>头的预处理指令，并删除注释。
- **具体动作：**
  - 展开头文件 <kbd>#include</kbd>
  - 宏替换 <kbd>#define</kbd>
  - 条件编译 <kbd>#ifdef</kbd>, <kbd>#if</kbd>等
- **IO：** <kbd>.cpp</kbd> ➔ <kbd>.i</kbd>

### 编译阶段 (Compilation)

- **核心任务：** 将 C++ 代码翻译成汇编语言
- **具体动作：** 词法分析、语法分析、语义分析、中间代码生成、代码优化、目标代码生成
- **性能特征：** 这是整个过程中最耗时、最吃 CPU 和内存的阶段。尤其是开启了高级优化（如 -O2, -O3）或大量使用 C++ 模板时，编译器负担极重
- **IO：** <kbd>.i</kbd> ➔ <kbd>.s</kbd>

### 汇编阶段 (Assembly)

- **核心任务：** 将汇编指令翻译成 CPU 能看懂的二进制机器码
- **IO：** <kbd>.s</kbd> ➔ <kbd>.o</kbd> 或 <kbd>.obj</kbd>

### 链接阶段 (Linking)

- **核心任务：** 将多个目标文件和库文件合并，生成最终的可执行程序
- **具体动作：** 
  1. 符号解析：把各个 <kbd>.o</kbd> 文件中缺失的函数/变量引用，与标准库或自己写的其他 <kbd>.o</kbd> 文件中的定义匹配起来
  2. 地址重定位：把代码段、数据段合并，计算出最终的内存绝对地址，修改机器码中的地址引用
- **链接分类：** 
  - **静态链接：** 把库代码直接拷进可执行文件
  - **动态链接：** 运行时再去加载动态库
- **IO：** <kbd>.o</kbd> ➔ ELF格式无后缀(Linux)/<kbd>.exe</kbd>(Windows)
