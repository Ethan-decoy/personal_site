# 指针与内存模型

> 费曼法第一篇：指针、引用、栈与堆、内存布局。目标是用自己的话讲清楚，而不只是"有印象"。

## 指针到底是什么

指针就是一个变量，存的值是另一个变量的**内存地址**。

```cpp
int x = 42;
int* p = &x;   // p 存的是 x 的地址
*p = 100;      // 通过 p 修改 x 的值，x 现在是 100
```

关键点：

- `&` 是取地址符，`*` 是解引用符
- 指针本身也有地址（`&p`），它是一个独立的变量
- 指针有类型，类型决定了解引用时读多少个字节、如何解释这些字节

## 指针 vs 引用

引用是指针的语法糖，但编译器层面的约束更强：

| | 指针 | 引用 |
|---|---|---|
| 可以为空 | 可以（`nullptr`） | 不行，必须绑定到一个对象 |
| 可以重新绑定 | 可以（`p = &y`） | 不行，初始化后不能改 |
| 有自身内存 | 有（存地址的变量） | 标准不保证，通常实现为指针 |
| 需要解引用 | 需要（`*p`） | 不需要（直接用 `r`） |

```cpp
int x = 1;
int& r = x;    // r 是 x 的别名，不能"重新赋值"为另一个变量
r = 5;         // 这是修改 x 的值，不是改变 r 的绑定

int* p = &x;
p = nullptr;   // OK，指针可以置空
```

面试常问：引用和指针的底层区别是什么？—— 引用在声明时必须初始化且不能改变绑定，编译器会在编译期强制检查；指针没有这些限制。

## 栈（Stack）vs 堆（Heap）

### 栈

- 编译器自动管理，函数调用时分配，返回时自动释放
- 速度快，空间小（通常几 MB）
- 局部变量、函数参数都在栈上
- 栈溢出 = 递归太深或局部数组太大

```cpp
void foo() {
    int a = 10;       // 栈上分配
    int arr[1000];    // 栈上分配，太大可能栈溢出
} // a 和 arr 在 foo 返回时自动销毁
```

### 堆

- 程序员手动管理（`new`/`delete`）或智能指针管理
- 空间大，但分配/释放比栈慢
- `new` 返回的是堆上对象的地址（指针）

```cpp
void bar() {
    int* p = new int(42);   // 堆上分配
    delete p;               // 必须手动释放，否则内存泄漏
} // p 这个指针变量本身在栈上销毁，但它指向的堆内存不受影响
```

### 经典陷阱：返回栈上局部变量的地址

```cpp
int* bad() {
    int x = 10;
    return &x;   // 危险！x 在 foo 返回后已销毁，p 变成悬垂指针
}
```

编译器通常会警告，但有些情况下不报。这是面试高频题。

## C++ 内存布局

一个运行中的 C++ 程序，内存分为以下几个区域：

| 区域 | 内容 | 生命周期 |
|------|------|----------|
| 代码区（Text） | 编译后的机器指令 | 程序整个运行期间 |
| 静态/全局区（Data/BSS） | 全局变量、static 变量、常量字符串 | 程序整个运行期间 |
| 栈（Stack） | 局部变量、函数参数、返回地址 | 函数调用期间 |
| 堆（Heap） | `new`/`malloc` 分配的内存 | 手动管理或智能指针管理 |

```cpp
int g = 10;              // 静态/全局区
const char* s = "hello"; // "hello" 在只读数据区，s 指针在栈上

void func(int param) {   // param 在栈上
    static int count = 0; // count 在静态区，不是栈上
    int local = 5;        // local 在栈上
    int* heap = new int;  // heap 指针在栈上，指向的 int 在堆上
}
```

> `static` 局部变量经常被问到：它和普通局部变量的区别是什么？—— 普通局部变量每次进入函数重新创建、退出销毁；`static` 局部变量只初始化一次，存在整个程序运行期间，存储在静态区而非栈上。

## 悬垂指针 vs 野指针

- **悬垂指针（Dangling Pointer）**：指向已被释放的内存。`delete p` 后没把 `p` 置 `nullptr`，`p` 就是悬垂指针。
- **野指针（Wild Pointer）**：未初始化的指针，值是随机的，解引用直接 UB。

```cpp
int* wild;       // 野指针，值是垃圾数据
delete wild;     // UB

int* p = new int(1);
delete p;
// p 现在是悬垂指针，*p = 5; // UB
```

防御写法：`delete` 后立刻 `p = nullptr`。不过智能指针时代不鼓励手动 `new`/`delete` 了。

## 智能指针速览

| 类型 | 语义 | 拷贝 | 适用场景 |
|------|------|------|----------|
| `unique_ptr` | 独占所有权 | 不可拷贝，只能移动 | 大多数情况的默认选择 |
| `shared_ptr` | 共享所有权（引用计数） | 可拷贝 | 多个所有者共享同一对象 |
| `weak_ptr` | 不拥有，观察 | 不可单独使用 | 打破 `shared_ptr` 循环引用 |

```cpp
auto up = std::make_unique<int>(42);   // 独占，离开作用域自动释放
auto sp1 = std::make_shared<int>(42);  // 引用计数 = 1
auto sp2 = sp1;                         // 引用计数 = 2
// sp1 和 sp2 都有效，最后一个销毁时才释放堆内存
```

> `make_unique`/`make_shared` vs 直接 `new`？—— `make_*` 是异常安全的：`f(unique_ptr<A>(new A), unique_ptr<B>(new B))` 中如果第二个 `new` 抛异常，第一个可能泄漏；而 `make_*` 不会有这个问题。
